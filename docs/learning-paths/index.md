# Learning Paths

These paths turn the 54-project catalog into sequenced routes. Each path is intentionally short enough to finish, but broad enough to build a portfolio story.

## Paths

| Path | Best for | Outcome |
|---|---|---|
| [Beginner](beginner.md) | New students | Linux, basic cloud app delivery, first CI/CD, and a local security demo |
| [Docker/Kubernetes](docker-kubernetes.md) | Container and cluster practice | Docker packaging, Kubernetes manifests, GitOps, microservices, and progressive delivery |
| [Terraform/IaC](terraform-iac.md) | Infrastructure builders | VPCs, EKS, ECS, serverless, OpenTofu, and validation-first infrastructure work |
| [CI/CD](cicd.md) | Delivery engineers | GitHub Actions, Jenkins, GitLab CI, Azure DevOps, AWS CodePipeline, and promotion workflows |
| [DevSecOps](devsecops.md) | Security-minded builders | SAST, container scanning, secrets hygiene, SBOMs, signing, and secure pipeline design |
| [AWS](aws.md) | Cloud portfolio work | AWS networking, containers, EKS, ECS, serverless, monitoring, and cost-aware cleanup |
| [Platform Engineering 2026](platform-engineering-2026.md) | Modern platform narrative | GitOps, observability, supply chain security, OpenTofu, progressive delivery, and flagship EKS |

## How to Use a Path

1. Read the path goal and complete the projects in order.
2. Use the [project picker](../catalog/project-picker.md) when you need a substitute with lower cost or different tooling.
3. Run the validation command from each `project.yaml` before deployment.
4. Keep a lab journal with setup notes, screenshots, failure modes, and cleanup proof.
5. End each path by writing one portfolio case study from the strongest project.
