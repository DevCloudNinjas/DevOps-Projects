# AWS Learning Path

This path is for students building an AWS-heavy portfolio. Watch cost risk closely and destroy cloud resources after every lab.

## Sequence

| Step | Project | Why it comes here |
|---|---|---|
| 1 | [02 - Scalable AWS VPC Architecture](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-02-aws-vpc-architecture) | Learn AWS network building blocks before deployment. |
| 2 | [04 - Django Application on AWS ECS and ECR](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-04-django-aws-ecs) | Package and push an app toward AWS containers. |
| 3 | [11 - Two-Tier AWS Infrastructure with Terraform](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-11-aws-2tier-terraform) | Provision a classic app infrastructure pattern. |
| 4 | [34 - Node CI/CD to ECS with Terraform and GitHub Actions](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-34-node-cicd-ecs-terraform-gha) | Combine ECS, Terraform, Docker, and GitHub Actions. |
| 5 | [42 - Serverless REST API with DynamoDB](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-42-serverless-api-dynamodb) | Add serverless architecture to the portfolio. |
| 6 | [48 - Terraform AWS EKS Provisioning](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-48-terraform-aws-eks) | Finish with a flagship Kubernetes infrastructure build. |

## Portfolio Prompt

Explain when you would choose EC2, ECS Fargate, EKS, or Lambda. Tie the answer to cost, operations burden, deployment speed, and team maturity.
