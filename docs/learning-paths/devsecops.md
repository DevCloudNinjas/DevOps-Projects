# DevSecOps Learning Path

This path is for students who want to add security checks to delivery without treating them as separate paperwork.

## Sequence

| Step | Project | Why it comes here |
|---|---|---|
| 1 | [44 - DevSecOps 101 Spring Boot Pipeline](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-44-devsecops-101) | Start with a focused pipeline and scanner vocabulary. |
| 2 | [40 - Kubernetes Dashboard with Trivy](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-40-k8s-dashboard-trivy) | Practice local vulnerability visibility. |
| 3 | [53 - Supply Chain Security Lab](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-53-supply-chain-security-lab) | Add SBOMs, image scanning, and signing concepts. |
| 4 | [09 - Netflix Clone DevSecOps CI/CD with Monitoring](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-09-devsecops-netflix-clone) | Study a fuller Jenkins-based security pipeline. |
| 5 | [35 - Progressive DevSecOps Pipeline Series](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-35-devsecops-pipeline-series) | Walk through multiple stages of maturity. |
| 6 | [45 - Jenkins CI/CD with Argo CD and Vault](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-45-jenkins-cicd-argocd-vault) | Finish with secrets management and GitOps delivery. |

## Portfolio Prompt

Create a security evidence bundle with scan output, false-positive notes, secret handling notes, SBOM or signing proof, and one remediation decision.
