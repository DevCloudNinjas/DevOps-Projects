# Docker/Kubernetes Learning Path

This path is for students who want hands-on confidence with images, manifests, services, local clusters, GitOps, and delivery patterns.

## Sequence

| Step | Project | Why it comes here |
|---|---|---|
| 1 | [04 - Django Application on AWS ECS and ECR](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-04-django-aws-ecs) | Start with a containerized app before cluster concerns. |
| 2 | [05 - Java App with Docker, Jenkins, and Kubernetes](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-05-docker-jenkins-k8s) | Connect image builds to Kubernetes deployment. |
| 3 | [08 - 2048 Game on Amazon EKS](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-08-2048-game-eks) | Practice a simple workload and service shape. |
| 4 | [41 - Online Boutique Microservices Assignment](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-41-online-boutique-microservices) | Move from one app to a microservices topology. |
| 5 | [50 - ArgoCD GitOps Home Lab](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-50-argocd-gitops-home-lab) | Learn declarative sync and drift detection. |
| 6 | [54 - Progressive Delivery Home Lab](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-54-progressive-delivery-home-lab) | Add canary rollout behavior after GitOps fundamentals. |

## Portfolio Prompt

Diagram the path from Dockerfile to running pod. Call out image tags, deployment manifests, service exposure, rollout strategy, and rollback behavior.
