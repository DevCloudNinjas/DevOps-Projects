# Terraform/IaC Learning Path

This path is for students who want to build infrastructure safely, read plans carefully, and explain cloud architecture from code.

## Sequence

| Step | Project | Why it comes here |
|---|---|---|
| 1 | [02 - Scalable AWS VPC Architecture](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-02-aws-vpc-architecture) | Learn network vocabulary before provisioning. |
| 2 | [11 - Two-Tier AWS Infrastructure with Terraform](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-11-aws-2tier-terraform) | Practice a smaller Terraform-backed architecture. |
| 3 | [52 - OpenTofu AWS Free-Tier Lab](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-52-opentofu-aws-free-tier-lab) | Compare Terraform-compatible tooling with cost awareness. |
| 4 | [43 - Scalable Web App on ECS Fargate with Terraform](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-43-ecs-fargate-terraform) | Apply IaC to managed container infrastructure. |
| 5 | [37 - Provision EKS Cluster with Terraform](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-37-eks-terraform-provision) | Move to Kubernetes control plane provisioning. |
| 6 | [48 - Terraform AWS EKS Provisioning](https://github.com/DevCloudNinjas/DevOps-Projects/tree/master/project-48-terraform-aws-eks) | Finish with a flagship EKS IaC case study. |

## Portfolio Prompt

Attach a sanitized plan summary and explain what each major resource group does, what it costs, how state is protected, and how cleanup is verified.
