# Flagship Docs

Flagship docs are the best starting points for complete DevOps workflows in this repository. They are useful examples for structure, validation, and security hardening.

Recommended starting points:

- [project-28-openai-chatbot-eks](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-28-openai-chatbot-eks): EKS, Jenkins, Docker, Terraform, and application security hardening.
- [project-31-cloud-native-monitoring](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-31-cloud-native-monitoring): Python service packaging, containerization, and monitoring workflow.
- [project-33-node-cicd-eks-gha](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-33-node-cicd-eks-gha): Node.js, GitHub Actions, Docker, and Kubernetes deployment flow.
- [project-37-eks-terraform-provision](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-37-eks-terraform-provision): EKS provisioning with Terraform.
- [project-38-docker-terraform-3tier](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-38-docker-terraform-3tier): Docker plus Terraform for a three-tier architecture.
- [project-46-eks-cluster-terraform-advanced](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-46-eks-cluster-terraform-advanced): advanced EKS cluster composition.
- [project-48-terraform-aws-eks](https://github.com/DevCloudNinjas/DevOps-Projects/tree/main/project-48-terraform-aws-eks): Terraform-managed EKS deployment examples.

Before updating a flagship project, check the relevant [security baseline](../security-baselines/index.md) and the [IaC/Kubernetes runbook](../iac-kubernetes-solid-runbook.md).
